## 一、用户故事

### US-BOOK-01：看到一本有交互感的立体书

> **作为**一个访问记忆池页面的用户，  
> **我想**看到一本有立体感的书，鼠标悬停时封面微微翻开，  
> **以便**感受到这个组件是"活的"，而不是一张静态插图。

**优先级**：P0（这是整个组件的存在意义）  
**阶段**：独立 UI 组件，不与 Pond 数据层绑定

---

## 二、组件结构说明

书本由以下零件组成，在 Z 轴上有明确的层叠顺序：

| 零件 | 说明 | Z 轴位置 |
|------|------|----------|
| 后封面 | 最底层，始终静止不动 | 最低 |
| 书页 | 一叠，比封面略小，右侧和底部可见纸张层叠纹理 | 中间 |
| 封面内侧 | 翻开后可见，颜色/样式与外侧不同 | 前封面背面 |
| 前封面 | 硬壳，有厚度，hover 时绕书脊翻开 | 最高 |
| 书脊 | 左侧窄条，连接前后封面，始终静止 | 左侧固定 |

## 三、验收标准

### 静止状态（默认）

| ID | 验收标准 | 优先级 |
|----|----------|--------|
| AC-BOOK-01 | 书本可见，呈现出有厚度的立体感，而不是一张扁平图形 | P0 |
| AC-BOOK-02 | 可辨识出书脊（左侧窄条）、前封面、书页侧面（右侧和底部可见纸张层叠纹理） | P0 |
| AC-BOOK-03 | 前封面与书页之间在视觉上有层次差异（前封面在上方，书页在下方） | P0 |
| AC-BOOK-04 | 静止状态下封面完全合上，看不到内侧 | P0 |

### Hover 状态（鼠标悬停）

| ID | 验收标准 | 优先级 |
|----|----------|--------|
| AC-BOOK-05 | 鼠标进入书本区域时，前封面绕书脊（左侧边缘）向外翻开，翻开角度约 25-30 度 | P0 |
| AC-BOOK-06 | 翻开后可以看到封面内侧（颜色/样式与封面外侧明显不同） | P0 |
| AC-BOOK-07 | 翻开后可以看到书页顶层（在封面内侧下方） | P0 |
| AC-BOOK-08 | 书脊、后封面、书页在 hover 过程中保持静止不动 | P0 |

### 动画行为

| ID | 验收标准 | 优先级 |
|----|----------|--------|
| AC-BOOK-09 | 翻开动画有弹性感：封面翻到目标角度后有轻微超出再回弹，而不是匀速到位 | P0 |
| AC-BOOK-10 | 鼠标离开时封面自然合上，合上动画同样带弹性感 | P0 |
| AC-BOOK-11 | 翻开和合上动画时长均约 0.5-0.6 秒 | P0 |
| AC-BOOK-12 | 连续快速进出（鼠标反复 hover/leave）时动画不出现闪烁或卡顿 | P1 |

### 组件独立性

| ID | 验收标准 | 优先级 |
|----|----------|--------|
| AC-BOOK-13 | 组件不依赖任何 Zustand store、任务数据、或记忆池逻辑 | P0 |
| AC-BOOK-14 | 组件接受基础外观 props（封面颜色、书脊颜色、内侧颜色），方便后续复用 | P1 |
| AC-BOOK-15 | 组件可在任意页面单独渲染，用于独立演示 | P0 |

## 四、非功能需求

| 类别 | 要求 | 优先级 |
|------|------|--------|
| 动画帧率 | 翻页动画在移动端（iPhone 12）不掉帧（≥ 60fps） | P1 |
| 可访问性 | 组件根元素有 `aria-label`（如"书本"） | P1 |
| 浏览器支持 | iOS Safari 16+，Chrome 110+（依赖 CSS 3D transform） | P0 |

## 六、交给 Designer 的问题

在 Developer 动工前，Designer 需要输出 `docs/design/book-card.md`，回答：

1. 书本的默认尺寸（宽 × 高 × 厚度）
2. 封面外侧 / 内侧 / 书脊 / 书页侧面 的颜色和材质（是否使用 liquid glass 效果？）
3. 书页侧面纹理的实现方式（多层 div 叠加 or CSS 渐变模拟）
4. 翻开动画的精确 spring 参数（stiffness / damping / 超出角度）
5. 封面转轴位置的精确偏移量（`transform-origin` 的 x 值）
